# 汎用フォールバックセット v1.0

基準:`MVP測定仕様書_v1.0_改訂1.md`
用途:共通タイプ質問と比較質問で単一タイプを確定できなかった低確信度ルート

## 1. 設計原則

- 特定タイプの欲望を前提にしない
- タイプが不明でも、欲求・希望・不満を外へ出す過程は測定する
- タイプ固有レポートの代替ではなく、指標レベルの誠実な結果を返す
- 全ルート最短39問を維持する
- 低確信度を「失敗」ではなく、複数動機または場面切り替えとして扱う

## 2. 汎用出し方4問

5段階:1=全く当てはまらない〜5=とても当てはまる。

| ID | 極性 | 質問 |
|---|---|---|
| GE01 | 表出+ | 自分が望んでいることを、周囲へ言葉で伝えることが多い |
| GE02 | 表出− | 本当に欲しいものほど、表には出さず自分の中にしまっておく |
| GE03 | 表出+ | 納得していないことがあると、態度や行動に表れやすい |
| GE04 | 使い分け | 相手や場面によって、自分の希望を見せるか隠すかを意識的に変えている |

```text
genericExpr = GE01 + (6 - GE02) + GE03
genericSwitch = GE04

外へ出す: genericExpr >= 12
内へ抱える: genericExpr <= 7
中間: 8..11
  GE04 >= 4: 使い分け傾向
  GE04 <= 3: 方向未確定、expressionConfidence=low
```

## 3. 汎用ズレ5ペア+確認1ペア

HとPをともに「外へ出したい・外へ出している度」1〜5へ揃える。

| ID | H:本音 | P:実際の振る舞い |
|---|---|---|
| GZ1 希望 | 本音では、自分の希望をもっと優先してほしいと思うことがある | 実際に、自分の希望を相手へ言葉で伝えている |
| GZ2 異論 | 本音では、納得できないときは違うと言いたい | 実際に、納得できないときは異論を示している |
| GZ3 境界 | 本音では、無理な頼みや負担は断りたい | 実際に、難しい依頼には断る・条件を伝える行動を取っている |
| GZ4 貢献 | 本音では、自分の努力や貢献に気づいてほしい | 実際に、自分が行ったことを必要な相手へ伝えている |
| GZ5 援助 | 本音では、困ったときは誰かに助けてほしい | 実際に、困ったときは助けを求めている |
| GZ6 確認 | 本音では、自分の都合や余裕も尊重してほしい | 実際に、予定や負担の調整を相手へ申し出ている |

GZ1〜GZ5で通常のズレ3指標を算出し、方向不明瞭時のみGZ6を追加する。

## 4. 低確信度パターンCの再計算

### タイプ
- baseTypeScores:勝5/つ4/読2/軸1
- 比較質問:勝×つが2対2
- resolvedType:単一確定せず、勝ち筋×つながりを併記
- typeConfidence:low

### 出し方
- GE01=3/GE02=3/GE03=3/GE04=5
- genericExpr=9
- 表示:使い分け傾向
- expressionConfidence:medium

### ズレ
- GZ1:H2/P4 → +2
- GZ2:H4/P2 → −2
- GZ3:H3/P4 → +1
- GZ4:H4/P2 → −2
- GZ5:H3/P3 → 0
- M=2、pos=2、neg=2、active=4
- 判定:反転型
- gapConfidence:high

### 表示上の制約
- 「勝ちたい本音を隠している」など単一タイプ前提の説明は禁止
- 「希望や不満を強く出す場面と、抑える場面が切り替わっている」と記述する
- 切り替え条件は今回測定していないため、相手・勝算・関係性などを断定しない

## 5. 低確信度結果文の基準

### 直接測定
「自分の希望を出すかどうかは、相手や場面で変えているという回答が出ています。」

### 算出
「本音より強く出す場面と、本音を抑える場面が同じくらい見られます。」

### 根拠付き推定
「一つの出し方に固定されず、状況に応じて前へ出る反応と引く反応を使い分けやすい可能性があります。」

### 禁止
- 二重人格・気分屋などの否定的ラベル
- 判定失敗・精度不足というユーザー向け表現
- 切り替えの原因や領域の断定
