# 防衛反応7問 バイアス点検 v1.0

基準:`MVP測定仕様書_v1.0_改訂1.md`
対象:全タイプ共通の防衛反応7問
目的:特定タイプへの語彙・魅力度・社会的望ましさの偏りを確認し、横展開前の共通バンクを確定する

## 1. 総合判定

現行7問は、初版にあった「勝つ・負ける・仲間・意味・理解」などタイプ固有語への偏りが概ね除去されており、4タイプ共通バンクとして使用可能。

ただし、7問に対して8カテゴリを扱うため、単純最頻だけでは同率が起きやすい。これはタイプ偏りではなく測定密度の問題である。MVPでは設問を増やさず、出力条件を厳しくする。

## 3. 選択肢の社会的望ましさ

現行では「分析」「小さな成果を積む」が健全に見えやすく、「迎合」「自責」「固まる」が否定的に見えやすい。全選択肢を同程度に選びやすくするため、結果の善悪ではなく、本人の内側で起きる自然な対処として表現する。

### 執筆ルール

- 「正しい」「冷静」「前向き」など評価語を付けない
- 選択肢の長さをそろえる
- 一つだけ具体的で、一つだけ抽象的な状態にしない
- 迎合は「関係修復」、分析は「原因究明」など美化された表現にしない
- 自責は「反省」、感情遮断は「切り替え」など肯定語に置換しない

## 4. 確定する防衛7問

現行の改訂1に記載されたD1〜D7を採用する。カテゴリ定義は以下で固定する。

| コード | 名称 | 観察可能な対処行動 |
|---|---|---|
| CT | 反撃 | 相手へ反論・追及・対抗する方向へ動く |
| PR | 証明 | 行動や成果によって評価・状態を立て直そうとする |
| DS | 距離 | 接点や刺激を減らし、自分を守る |
| AP | 迎合 | 自分の希望や不満を抑え、相手の期待へ合わせる |
| AN | 分析 | 感情処理より先に、事実・原因・構造を整理する |
| SB | 自責 | 原因や矛先を自分へ向ける |
| NU | 感情遮断 | 感情への接続を弱め、機械的に日常を続ける |
| FR | 固まり | 思考や行動が一時的に停止する |

## 5. 集計ルールの修正

7問×8カテゴリでは同率が自然に起こるため、以下とする。

```text
counts = 各カテゴリの選択回数
maxCount = 最大回数
leaders = maxCountと同数のカテゴリ一覧

if maxCount >= 3 and leaders.length == 1:
  第一防衛 = leaders[0]
  defenseConfidence = high
elif maxCount == 2 and leaders.length == 1:
  第一防衛 = leaders[0]
  defenseConfidence = medium
elif maxCount == 2 and leaders.length == 2:
  併記防衛 = leaders[0] + leaders[1]
  defenseConfidence = low
else:
  単一の第一防衛は出さない
  表示 = 「複数の守り方を場面で使い分ける傾向」
  defenseConfidence = low
```

- 2回が3カテゴリ以上並立した場合は、第一防衛を出さない
- 次点は、第一防衛が単独成立し、かつ次点が2回以上の場合のみ表示
- 存在意義D7の1回答だけで独立ラベルを作らない
- 防衛カテゴリに優劣・成熟度を付けない

## 6. 横展開時の確認事項

残り3タイプの結果文を書く際も、防衛の説明はタイプ固有の欲望と機械的に結びつけない。同じANでも、タイプにより守ろうとしているものは異なる可能性があるが、それはレベル3の推定として記述する。

例:
- 直接測定:「失敗後に原因を整理する反応を選んでいます」
- 根拠付き推定:「感情を表に出す前に、理解できる形へ変えることで自分を守りやすい可能性があります」
- 禁止:「読み解きタイプだから分析防衛になります」
