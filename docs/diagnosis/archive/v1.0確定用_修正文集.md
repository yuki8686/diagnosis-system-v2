## 修正1:§6 有料版で出す内容(4項・5項の差し替え)

【旧】
4. ズレの詳細(どの本音を、どんな振る舞いで隠し、恋愛・仕事・人間関係のどこで繰り返すか)
5. 防衛反応の詳細(3シナリオ+存在意義の否定の反応が各場面でどう出るか)

【新】
4. ズレの詳細(どの本音を、どんな振る舞いで隠しているか=算出結果。恋愛・仕事・人間関係での現れ方は「似た場面ではこう出ることがある」という可能性としての提示に限定し、「あなたは恋愛でこうしている」という領域断定はしない。領域断定は該当する精密版モジュール受診後にのみ解放)
5. 防衛反応の詳細(各シナリオで選択した反応=直接測定。反応の継続や領域別の現れ方は可能性としての提示。存在意義の否定は検出1問のため、反応の型までを測定として扱い、それ以降は可能性として記述)

## 修正2:§4 低確信度ルート(差し替え)

【旧】汎用出し方(2)/比較質問なしで到達:37問/比較質問ありで到達:41問(最大43問)

【新】
【低確信度ルート】39〜45問
  汎用出し方(4) → 防衛反応(7) → ズレ:汎用(10) → 使いこなし:上位2タイプ各3問(6)
  比較質問なしで到達:12+4+7+10+6=39問
  比較質問ありで到達:12+4+4+7+10+6=43問
  追加は信頼性再確認+2問のみ許可(最大45問)

【追記】汎用出し方4問の設計:タイプ非依存の表出傾向を、欲望一般(勝ち・つながり・理解・理想のいずれにも読める文脈)で聞く4問。タイプ別4問より精度が落ちる前提を内部フラグで保持する。

これにより「どのルートでも最短39問」が全ルートで成立する。§3冒頭の「標準39問を最短とする」は「全ルート共通の最短は39問」に修正。

## 修正3:§3 追加確認質問(予算ルールの差し替え)

【旧】予算原則:追加は優先順位順に消化し、48問到達後は打ち切る。

【新】予算原則:追加は2問単位(比較質問のみ2問または4問)で優先順位順に消化する。次の単位を加えると48問を超える場合、その単位は発動せず、当該の不確かさはレポート表現の弱め(断定→傾向表現への自動切り替え)で吸収する。予算上限は48問、この運用での実到達最大は47問(例:比較4+出し方2+使いこなし2=39+8=47、ズレ確認は表現側で吸収)。

## 修正4:§3への追記(タイプ不適合フラグ)

【追記・新設】
### タイプ不適合フラグ(誤判定の一方通行防止)
タイプ確定後のブロックは当該タイプ前提のため、序盤の誤判定は下流で自動修正されない。最低限の反証機構として以下を定める。
- タイプ別ズレの本音フレーム5問の平均が2.0以下(仮閾値)の場合、想定した欲望が本人に共鳴していないと判断し「タイプ不適合フラグ」を立てる
- フラグ時の処理:診断は中断せず最後まで実施。結果はサブ傾向を併記した弱め表現に自動切り替え。フィードバックの「違和感があった項目」回答と紐付けてログ保存し、共通タイプ質問の改修材料とする
- 再ルーティング(途中でタイプを変えて質問を出し直す)はMVPでは行わない

## 修正5:§3への追記(タイプ判定の数値ルール・仮閾値)

【追記・新設】
### タイプ判定ルール(v1.0仮閾値)
- 配点:共通12問は各1点を選択タイプに加算。合計12点。T1≧T2≧T3≧T4とする
- 確定:T1−T2≧3
- 僅差:T1−T2が0〜2(同点含む)→上位2タイプ比較質問を発動
- 団子:T1−T3≦2、またはT1≦4(平坦プロファイル)→比較質問を経ず低確信度ルートへ直行
- 比較質問:まず2問。2問とも同一タイプ→確定。1対1→追加2問(計4問)。4問中3問以上同一→確定。2対2→低確信度ルートへ
- サブ傾向表示:T2≧3点の場合に表示
- 確信度:内部3段階。高=T1−T2≧3または比較で確定/中=比較を経て確定/低=低確信度ルート
- 上記の数値(3点、2点、平均2.0等)はすべて仮閾値であり、リリース後に再検査一致率・フィードバック・不適合フラグ発生率で校正する。校正対象であることをログ設計に反映する(§12)

### 信頼性フラグ(仮閾値)
- 回答時間:1問あたりの中央値回答時間が1.5秒未満→高速フラグ
- 直線回答:同一位置の選択肢を8問連続→直線フラグ
- いずれも質問追加はせず、該当ブロックの表現弱めで吸収(既定原則の通り)

## 修正6:ズレ算出の疑似コード(§3ズレのスコアリング項の差し替え)

【旧】5ペアの乖離の単純平均ではなく、中央値(または最大・最小を除いた3ペアの平均)を採用。

【新】
### ズレ3指標の算出
前提:全ペアはスコアリングキーで極性を統一し、H(本音)・P(対人)とも「欲望を出している度」(1〜5)に揃えてから計算する。

```
各ペア i (i=1..5):
  d[i] = P[i] - H[i]        # 負=本音より抑えている / 正=本音より出している・演じている

大きさ M = median(|d[1..5]|)             # 0〜4
方向:
  neg = count(d[i] <= -1)
  pos = count(d[i] >= +1)
  zero = count(d[i] == 0)
一貫性 C:
  active = neg + pos
  if active == 0: C = null(ズレ小として扱う)
  else: C = max(neg, pos) / active        # 0.5〜1.0

判定:
  if M <= 1 and active <= 1:        → 「ズレ小」型
  elif neg >= 2 and pos >= 2:       → 「反転」型(場面によって切り替わる)
  elif C >= 0.75 and neg > pos:     → 「抑圧」型(本音を内に抑える)
  elif C >= 0.75 and pos > neg:     → 「演出」型(本音より大きく見せる)
  else:                             → 「方向不明瞭」→ ズレ確認質問(+1ペア)の発動条件。
                                       予算超過時は表現弱めで吸収

外れ値処理:Mに中央値を用いることで単一ペアの外れ値の影響を抑制。
           |d[i]|=4のペアが1つだけある場合は該当ペアを回答引用の候補にする
           (外れ値は誤答か、最も鋭い個別性かのどちらかであるため)
```

レポート表現への変換:
| 型 | 表現の骨子 |
|---|---|
| ズレ小 | 本音と振る舞いが近く、裏表のなさが信頼になりやすい |
| 抑圧 | 本音を内側に抑えて振る舞う場面が多い(課金フックの主対象) |
| 演出 | 本音より大きく・強く見せる場面が多い |
| 反転 | 場面によって出し方が切り替わりやすい(「ズレがない」とは書かない) |
| 方向不明瞭 | 断定を避け、傾向の可能性として短く記述 |

## 修正7:§5・§6への追記(一撃の根拠レベル規定)

【追記】
### 一撃の執筆規則
- 一撃は根拠付き推定(レベル3)として扱う。描写する行動の核は、直接測定したタイプの核(欲望の優先順位)の言い換えに限定する
- 具体的な場面(会議・LINE・買い物等)は例示であることを「たとえば」で明示する
- 文末は「〜ているはずです」「〜ていませんか」型とし、「〜します」「〜です」の断定形は使わない
- 例:「たとえば会議で自分の案が通らなかったとき——あなたは頷きながら、頭の中ではもう次の勝ち筋を探し始めているはずです」
- 低確信度ユーザーには併記形式の一撃テンプレートを使用(既定の通り)

## 修正9:境界文書「相性の扱い」の差し替え

【旧】双方のタイプと出し方から、自然に合いやすい部分、翻訳が必要な部分、すれ違いやすい論点を算出する。

【新】双方のタイプと出し方に、タイプ組み合わせ別のテンプレートを適用し、合いやすい部分・すれ違いやすい部分の見込みを推定する(根拠付き推定)。二人の実際の相互作用を測定したものではないため、表現は「〜しやすい」「〜になりやすい」に限定し、関係の現状や将来を断定しない。
