# AGENTS.md

## プロジェクト

本プロジェクトは、本能・欲望・欲望の出し方・振る舞い・ズレ・防衛反応・使いこなしを測定する診断システムver.2です。

基準仕様は `docs/diagnosis/` 内の最新版です。

## 最優先の基準

以下の順で参照してください。

1. `docs/diagnosis/MVP測定仕様書_v1.0_改訂5.md`
2. TypeScript実装内の型・スコアリング仕様
3. 質問バンク移植報告
4. 無料・有料結果テンプレート
5. その他の補助資料

資料間で矛盾がある場合は、勝手に推測して実装せず、矛盾箇所を報告してください。

## 診断上の重要原則

結果文は必ず以下の4段階に分けて扱ってください。

1. direct：直接測定
2. derived：組み合わせによる算出
3. inferred：根拠付き推定
4. possibility：可能性としての提示

測定していない内容を、本人の事実として断定しないでください。

恋愛、仕事、家族、友人などの領域別行動は、該当モジュールを受診していない限り可能性表現に限定してください。

## 質問バンク

物理質問バンクは163問です。

これはユーザーが全問回答するという意味ではありません。

ユーザーの回答数は以下です。

* 確定ルート：39〜47問
* 低確信度ルート：39〜45問
* ユーザー向け表示：「所要時間 約10分前後」
* ユーザー画面では総問数を表示しない

質問ID、選択肢とタイプの対応、逆転項目、ズレペア、極性、防衛カテゴリを勝手に変更しないでください。

変更が必要な場合は、変更理由と影響範囲を先に示してください。

## スコアリング

共通12問の得点と比較質問の得点は分離してください。

* baseTypeScores
* comparisonScores
* resolvedType

タイプ確信度、出し方確信度、ズレ確信度、防衛確信度、使いこなし確信度も別々に管理してください。

タイプ不適合フラグは、使いこなしの低さだけで立てないでください。

位置連打だけで回答の信頼性を低いと判定しないでください。

## 結果生成

無料結果は、タイプ一般論だけで生成しないでください。

有料結果には最低限、以下の個別データを使用してください。

* 最大ズレ場面
* ズレの方向・強さ・広がり
* 第一防衛または同率防衛
* 気づけている実感
* 活かせている実感
* 本人が実際に選んだ回答3か所以上
* 回答に基づく具体的な行動提案1つ

有料結果が無料結果の文章量追加にならないようにしてください。

## 表現上の禁止事項

以下を断定しないでください。

* 本人の本心
* 幼少期や過去の原因
* トラウマ
* 脳構造
* 心理疾患
* 現実の仕事成果
* 恋愛の成功
* 相手の感情
* 関係の将来
* タイプ間の優劣

## 開発方針

既存コードを確認せずに大規模な書き換えをしないでください。

作業前に以下を行ってください。

1. リポジトリ構造を確認
2. READMEと仕様書を読む
3. package.jsonのスクリプトを確認
4. 既存テストを実行
5. 変更対象と影響範囲を整理

作業後は以下を実行してください。

* 型チェック
* lint
* テスト
* 必要に応じてbuild
* git diff確認

テストを削除または弱めて通すことは禁止です。

## UI

UIを実装する場合も、内部の質問数は表示しないでください。

開始画面は以下を使用します。

* 所要時間：約10分前後
* 途中保存できます

進捗はSTEP表示とプログレスバーを使用してください。

## 作業報告

作業完了時は以下を報告してください。

* 確認したファイル
* 変更したファイル
* 実装内容
* テスト結果
* 未解決事項
* 次に進める作業

曖昧な「完了しました」だけで終わらせないでください。
